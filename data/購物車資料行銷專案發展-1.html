Chapter 3 購物車資料：行銷專案發展

在Kaggle上搜尋到的資料集，資料標題名稱為Instacart Market Basket Analysis，內容為超市顧客的訂單數據。 https://www.kaggle.com/datasets/psparks/instacart-market-basket-analysis

3.1 輸入資料與資料統整

3.1.1 輸入資料與合併

在資料包中總共有六個檔案，分別是：

  1. aisles.csv

  2. departments.csv

  3. order_products__prior.csv

  4. order_products__train.csv

  5. orders.csv

  6. products.csv

依序將檔案輸入到R studio 中

library(dplyr)

3.1.2 資料欄位說明

字段名稱 描述
user_id 顧客編號
order_id 訂單編號
aisle_id 商品所屬走道編號
aisle 商品所屬走道
department_id 商品所屬部門編號
department 商品所屬部門
product_id 商品編號
product_name 商品名稱
add_to_cart_order 表示商品在訂單過程中被加到購物車的順序，可以識別哪些商品在訂單過程中較早或較晚被加入購物車。
reordered 表示顧客是否在過去訂購過該商品。如果值是 0，表示商品是第一次被訂購。如果值是 1，表示商品在過去已被顧客購買過。
eval_set 表示訂單屬於哪個評估集合。有三種值：「prior」代表測試集之前的訂單，「train」代表訓練集的訂單，以及「test」代表測試集的訂單。
order_number 表示顧客的訂單編號。例如，如果這是顧客的第一個訂單，這個值將是 1。如果這是顧客的第二個訂單，這個值將是 2，依此類推。
order_dow 表示訂單是在一週的哪一天下的。值的範圍從 0 到 6，0 代表星期六，6 代表星期五。
order_hour_of_day 表示訂單是在一天中的哪個小時下的。值的範圍從 0 到 23，0 代表半夜12:00，23 代表晚上 11:00。
days_since_prior_order 表示從顧客上次訂單至此次訂單之間經過的天數。例如，如果這是顧客的第一個訂單，這個值將是 NaN（非數字）。如果這是顧客的第二個訂單，而且是在第一個訂單之後的第 7 天下的，這個值將是 7。

3.2 資料集群大小

在資料集中，訂單總數3,421,083筆資料中，包含prior, test, train

  1. prior（之前）：包含用戶之前的訂單數據。通常是最大的數據集，用於了解用戶的購買歷史和行為模式，也可以用來訓練機器學習模型，以預測用戶未來的購買行為或對商品的偏好。
  2. train（訓練）：訓練集用於建立或訓練機器學習模型。數據包含了用戶的訂單信息及其相應的輸出（例如，用戶是否再次購買某個商品）。模型在這個數據集上學習，調整其內部參數，以最佳方式對比輸入數據到預期輸出。
  3. test（測試）：測試集用來評估已經訓練好的機器學習模型的性能。這部分數據在模型訓練過程中是不可見的，以確保評估結果能公正地反映模型在處理未見過數據時的實際表現。使用測試集可以檢查模型是否過擬合訓練數據，以及它對新數據的泛化能力。
eval_set_counts <- orders %>%
  count(eval_set) %>%
  rename(評估集合 = eval_set, 訂單數量 = n)

3.3 發展問題

電商能夠透過資料分析預測消費者行為，購買Ａ商品後下一個可能想買的東西為何？

  1. 提升銷售和轉化率：通過向顧客推薦他們可能感興趣的商品，從而提升總體的銷售量和轉化率。
  2. 增強顧客體驗：精準的推薦可以提升顧客的購物體驗，提升顧客滿意度和忠誠度。
  3. 優化庫存管理：透過預測分析，商家可以更好地理解顧客的購買趨勢和偏好，減少過剩或缺貨的情況。
  4. 提高行銷效率：幫助了解最有可能對特定產品感興趣的顧客群體，使行銷更加精準
  5. 減少退貨率：如果顧客購買的商品更符合需求和偏好，退貨的可能性也會降低。

customer_loyalty <- average_repurchase %>%

(二) 重複購買率最高的前十名商品

緊接者好奇熱門品項的重複購買率是否也是最高的？ 計算每個產品的訂單總數和重複購買訂單數 先彙整出產品忠誠度data frame

product_loyalty <- complete_df %>%
  group_by(product_id, product_name,department) %>%
  summarise(
    total_orders = n(), 
    reordered_count = sum(reordered),

  )

計算重複購買率（重複購買訂單數/訂單總數）

product_loyalty <- product_loyalty %>%
